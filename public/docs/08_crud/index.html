Creando un CRUD

Realizar un CRUD

CRUD

Vamos a hacer una aplicación que premita interactuar con una tabla en la base de datos a través de la aplicaicón.

La palabra CRUD viene de Create Read Update Delete, es decir, que desde nuestra aplicación podamos hacer estas acciones sobre la tabla

Acciones necesarias
  • Creamos un proyecto nuevo
  • Crear la tabla
  • Poblar la tabla con valores
  • Crear un Modelo
  • Crear un Controlador de tipo recurso
  • Establecer las rutas y conocerlas
  • Escribir las acciones para cada método del controlador según la ruta solicitada
  • En cada solicitud crear la vista necesaria para visualizar información

Qué son los elementos

Clases especiales de laravel que creamos

Migration

Son clases anónimas clase nos va a permitir realizar acciones de tipo DDL sobre la base de datos
  • crear
  • modificar
  • borrar
  • tablas
  • índices, etc

Cada clase tiene dos métodos:

Método que creará elementos

Se ejecutará cuando ejecute una migración

Este método debería de elminar los elementos creados en up

Se ejecutará cuando ejecute un dehacer migración (con rollback o reset)

Cada migración va a ser una clase ubicada en 

./database/migrations/fecha_create_xxxx.php

Factory

Para dar 

fabricar o crear valores o filas de una determinada tabla

Las factorías crean valores, no los inserta en la tabla

Las factorías se ubican en 

./database/factory/NombreFactory.php

Seeder

Esta clase va a llamar a la 

factoría y cada fila que nos retorna la vamos a insertar en la tabla

Los seeder crean valores, no los inserta en la tabla

Las seeder se ubican en 

./database/seeder/NombreSeeder.php

<p>Como puede haber varios seeder, todos se van a ejecutar a través de DatabaseSeeder.php

Por ejemplo si tuviera varios seeder

Modelo

Es una clase que nos va a permitir 

interactuar con una determinada tabla de la base de datos de forma

directa

Para ello utilizaremos 

objetos y métodos

Eloquent

ORM

Las modelos se ubican en 

./app/Models/Modelo.php

Un modelo asume por defecto valores para interactuar con una tabla sin tener que especificar nada:

  • Nombre de la tabla igual que 
nombre del modelo en plural

alumnos
  • Clave en la tabla 
id (autoincrement )
  • Tiene los campos de maracas de tiempo: 
create_at update_at

Estos valores se pueden cambiar estableciéndolos en el modelo correspondiente
